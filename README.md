1.	Описание архитектуры системы.
Созданы два актора – «Producer» и «Consumer», они взаимодействуют через сервис «Kafka», за хранение данных отвечает БД Postgres.
Producer предоставляет REST API с end-point / metrics, который создает заказ на приобретение товара (Order), сохраняет его в БД и направляет сообщение в topic «metrics-topic» с помощью Кафки.
Consumer предоставляет REST API с end-points:
- «/metrics» - получает все метрики,
- «/metrics/{id}» – получает одну метрику по её id,
- «/metrics/between» - получает метрики между заданными датами.
Consumer получает указанные метрики у «metrics-topic» с помощью Кафки и сохраняет их в собственной БД. Далее из этой БД данные берутся для последующего анализа, сбора статистики и т.д. В качестве примера, сделан end-point «/metrics/between», который получает метрики между заданными датами, но возможно предложить и иные методы обработки.

2.	Инструкция по запуску и использованию системы. 
Для запуска системы требуется Кафка с портом 9092, а также БД PostgresSQL с портом 5432, в котором созданы две БД – «Metrics» (для Consumer) и «Orders» (для Producer).
Логин и пароль для БД – «postgres», «password», при необходимости можно заменить на иной.
Далее необходимо запустить проект Producer (на порту 8080), чтобы он создал topic. После этого запускается Consumer (на порту 8081).
Чтобы создать заказ (order), необходимо использовать end-point «metrics» и направить post-запрос на адрес localhost:8080/metrics с телом:
{
  "productName": "sugar",
  "orderingDate": "2024-04-02",
  "price": 70
}

Чтобы посмотреть метрики, необходимо использовать end-point «metrics» и направить get-запрос на адрес localhost:8081/metrics.
